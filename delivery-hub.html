Delivery flow (API + demos)

Customers place orders tied to a merchant user id. Merchants get SSE + optional email when a rider is not involved yet. After accept, all riders subscribed to SSE see the job; first claim wins. Riders update GPS so customers see live maps on track order.

  • Customer — place delivery order
  • Merchant — inbox + Accept
  • Customer — live SSE inbox

SSE streams: /api/v1/orders/stream/{merchant|rider|customer}?token=<jwt> · REST under /api/v1/orders/*
